React 在样式方面没有固定的意见。因此，在将 CSS 与 React 结合时，有很多选择。

从技术上讲，没有什么可以阻止你以“传统”的方式做事。你可以保留一堆 CSS 文件，并在你的 index.html 中使用 \<link\> 标签将它们包含进来。

但老实说，如果你想在使用 React 时获得最佳体验，还有更好的方法。

组件的核心思想是每个组件都是一组标记（在 JSX 中）、逻辑（在 JS 中）和样式（在 CSS 中）。我们的 Button 组件应该“拥有”与该组件相关的所有样式！

当我们以这样的方式结构化我们的代码时，会发生一些神奇的事情。使用 CSS 变得简单多了。

编写 CSS 的传统方式涉及编写许多不同的选择器来定位相同的元素。这导致了一种“军备竞赛”，每个选择器都需要变得越来越强烈，以覆盖现有的样式。

在大型项目中编写 CSS 一直是个令人沮丧的问题，并且有很多解决方案被提出。其中最受欢迎的一种是 BEM（块元素修饰符），一种命名方法论。

如果 100%遵循，BEM 确实使 CSS 的工作变得容易。不幸的是，人性使得在大型项目中，尤其是有许多开发者时，很容易偏离轨道。遵循这种方法论需要超人的自律。

当 React 逐渐崭露头角时，社区意识到有机会改善现状。在随后的几年中，发布了许多不同的工具，包括 styled-components、emotion、vanilla-extract、stitches 等。

这些工具“嵌入”了范围，自动为我们生成独特的选择器。
